# 뱅카우 프로토타입

## 이 프로젝트는
뱅카우 서비스 기능의 플로우를 빠르게 확인하기 위한 인터랙티브 프로토타입이다.
디자인 완성도가 아니라 흐름과 인터랙션 확인이 목적이다.

## 핵심 원칙
- 속도 우선. 빠르게 만들고, 써보고, 수정한다.
- 피그마 와이어프레임의 구조와 플로우를 정확히 따른다.
- 비주얼은 단순하게. 플로우가 돌아가면 된다.
- 없는 UI는 그 자리에서 바로 만든다.

## 디자인
- tokens.css에 있는 토큰은 쓴다. 없는 값은 하드코딩한다.
- tokens.css에 새로운 토큰을 임의로 추가하지 않는다.
- components/에 있는 공용 컴포넌트는 재활용한다.
- components/에 새로운 컴포넌트를 임의로 추가하지 않는다.
- 없는 UI는 화면 코드 안에서 직접 만든다.
- 모바일 화면 좌우 패딩은 16px 고정.

## 피그마
- 피그마 URL을 주면 get_design_context로 구조를 파악하고 플로우를 구현한다.
- get_design_context가 반환한 코드는 참고만 한다. 그대로 쓰지 않는다.
- 구조와 플로우만 파악하고, 코드는 직접 간단하게 작성한다.

## 파일 구조
- components/ → 공용 컴포넌트 (내가 직접 추가한 것만)
- pages/v1/, v2/ → 버전별 프로토타입
- 각 버전의 index.jsx가 해당 버전 엔트리

## 확인 및 공유
- 데스크톱: npm run dev → localhost에서 직접 클릭하며 확인
- 모바일(나): 같은 와이파이에서 Network URL로 폰 접속
- 팀원 공유: Vercel 배포 URL 공유 (git push하면 자동 업데이트)
  - /v1, /v2 등 버전별 URL로 접근 가능
  - 예: bankcow-lottery.vercel.app/v1

## 새 화면 만들 때
1. 피그마 URL 받으면 구조 파악
2. 플로우 구현
3. 끝. 과하게 다듬지 않는다.
